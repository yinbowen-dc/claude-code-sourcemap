# 认证与安全

## 概述

多来源认证体系，以 OAuth2 + PKCE 为主要流程，API Key 作为备选。
macOS 使用 Keychain 安全存储，Linux 退回明文存储。

**相关文件：** `cli/handlers/auth.ts`、`commands/login/`、`commands/logout/`、
`bridge/jwtUtils.ts`、`bridge/workSecret.ts`、`bridge/trustedDevice.ts`、
`services/oauth/`、`schemas/`、`commands/mcp/xaaIdpCommand.ts`、
`commands/install-github-app/`

## 认证优先级层级（7 层）

```
1. ANTHROPIC_API_KEY 环境变量        → 直接 API Key（最高优先级）
2. ANTHROPIC_AUTH_TOKEN 环境变量      → Bearer Token 覆盖
3. apiKeyHelper shell 命令           → 从外部命令动态获取 Key
4. Claude AI OAuth Token（Keychain） → 交互式 OAuth 登录
5. Console OAuth Token（Keychain）   → Anthropic Console OAuth
6. Bedrock 凭证                      → AWS 凭证链
7. Vertex AI 凭证                    → Google Cloud 凭证（最低优先级）
```

## OAuth2 + PKCE 流程

```
1. 生成 PKCE code_verifier（随机字节，base64url 编码）
2. 推导 code_challenge = base64url(SHA256(verifier))
3. 构建含 challenge + state + scopes 的认证 URL
4. 打开浏览器到 Claude.ai / Console OAuth 端点
5. 本地重定向服务器捕获授权码
6. POST /oauth/token，带 code + verifier
7. 响应：{ access_token, refresh_token, expires_in }
8. 将 refresh_token 存入 Keychain
9. 在内存中缓存 access_token（过期时清除）
10. POST /profile → 将账号元数据存入 ~/.claude/config.json
```

### Token 刷新

- Access Token 在**过期前 5 分钟**刷新
- 在途去重：同时只有一个刷新请求（世代计数器）
- **跨进程陈旧检测**（通过文件 mtime）：
  ```typescript
  // 如果 config.json 被外部修改（另一个 Claude 实例刷新了 Token），
  // 从文件读取最新 Token，而不是使用内存中的过期缓存
  ```
- **出错时使用旧缓存：** Keychain 读取失败时返回上一个已知有效的 Token
  （防止 Keychain 偶发问题导致"未登录"级联错误）

### 401 在途去重

收到 401 时如果刷新已在进行中：
```typescript
// 在第一个 await 前同步声明在途槽位
// （防止笔记本唤醒时的"惊群效应"）
if (authRecoveryInFlight) {
  await authRecoveryInFlight
  return retry()
}
authRecoveryInFlight = refresh()  // await 前声明
```

## GitHub App 安装（install-github-app/）

创建 Claude GitHub Actions 工作流的多步向导：

1. 检查 GitHub CLI 已认证
2. OAuth 授权 GitHub 权限范围
3. 选择仓库
4. 检查已有密钥/工作流
5. 安装 GitHub App
6. **⚠️ 安全问题：** API Key 通过以下命令存储为 GitHub Actions 密钥：
   ```
   gh secret set ANTHROPIC_API_KEY --body "$key" --repo owner/repo
   ```
   执行期间 Key 出现在进程参数中（`ps aux` 可见）。

## 安全发现

| 问题 | 严重程度 | 详情 |
|------|---------|------|
| Linux 明文 Token 存储 | **高** | `~/.claude/.credentials.json` 未加密；TODO：libsecret |
| GitHub 密钥在进程参数中 | **中** | `gh secret set ANTHROPIC_API_KEY --body "$key"` 在 ps 中可见 |
| 账号元数据未加密 | **中** | 邮箱、组织名存储在 `~/.claude/config.json` |
| JWT 解码不验证签名 | **低** | 可接受：Token 来自可信服务器，仅用于检查过期时间 |
| 环境变量覆盖 OAuth（ANTHROPIC_API_KEY） | **低** | 故意设计：开发者覆盖用；已有文档说明 |

## 优秀安全模式

| 模式 | 说明 |
|------|------|
| PKCE 授权码挑战 | 防止授权码被拦截 |
| 401 在途去重 | 防止 Token 过期时的惊群效应 |
| 跨进程 mtime 检测 | 多个 Claude 实例不争夺 Token |
| 出错时使用旧缓存 | Keychain 偶发问题不导致用户退出登录 |
| 退出前刷新遥测 | 防止跨组织数据泄漏 |
| xaaIdp 仅 HTTPS | 防止中间人攻击窃取凭证 |
| xaaIdp 独立信任域 | MCP Token 不能升级为主 OAuth 权限 |
