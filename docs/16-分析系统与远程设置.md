# 分析系统与远程设置管理

## 概述

Claude Code 的分析系统采用双汇聚架构（Datadog + 内部），集成 GrowthBook A/B 测试，
并通过远程托管设置（remoteManagedSettings）实现组织级控制。

**相关文件：** `services/analytics/`、`services/remoteManagedSettings/`、
`services/api/grove.ts`、`services/growthbook/`

---

## 分析系统（analytics/）

### 双汇聚架构

```typescript
type AnalyticsSink = 'datadog' | 'firstParty'

// Datadog Sink：
// - 公开写入 Token（硬编码在源码中）
// - 仅接受预定义的事件模式
// - 提供实时监控仪表盘

// FirstParty Sink（内部）：
// - 发送到 Anthropic 内部数据管道
// - 包含更丰富的事件字段
// - 仅在认证用户会话中启用
```

### 🔥 Datadog 公开 Token 硬编码

```typescript
// Datadog 的日志写入 API Token 直接硬编码在源码中
// 这不是安全问题——Datadog 的设计就是允许公开写入 Token
// （与读取/管理 Token 不同）
// 任何人都可以向这个 Datadog 组织发送日志
// 但只有授权的分析师才能读取
const DATADOG_TOKEN = 'pub...'  // 公开写入专用
```

### 🔥 GrowthBook 企业代理主机名泄露

```typescript
// 企业客户可以配置自定义 GrowthBook 代理
// 代理 URL 包含企业内部主机名
// 这个主机名会出现在遥测事件中：

logEvent('tengu_growthbook_config', {
  proxyUrl: config.growthBookProxy,  // ← 暴露内部主机名
})

// 已知问题：安全审查时发现，但"低优先级"
// 企业部署时可能通过遥测泄露内部网络拓扑
```

### 双 Sink Kill-Switch

```typescript
// GrowthBook 标志：tengu_frond_boric
// 允许紧急关闭任一数据汇
// 失败开放设计：kill-switch 查询失败时，两个 sink 都继续工作

type SinkKillswitchConfig = {
  disableDatadog?: boolean
  disableFirstParty?: boolean
}
```

### AnalyticsMetadata 品牌类型

## GrowthBook A/B 测试集成

### 缓存模式

```typescript
type GrowthBookCacheMode = 'CACHED_MAY_BE_STALE' | 'FRESH'

// CACHED_MAY_BE_STALE：
// - 使用内存缓存，可能过期
// - 启动性能好（无网络延迟）
// - 但首次启动时缓存为空，返回默认值

// 历史教训（worktree 模式）：
// tengu_worktree_mode 使用 CACHED_MAY_BE_STALE 模式
// 首次启动缓存为空 → 返回 false
// 导致 --worktree 标志被静默丢弃
// 修复：改为始终启用（return true）
```

### 特性标志评估

```typescript
// getFeatureValue_CACHED(flagName, defaultValue)
// FRESH 模式：每次从 GrowthBook 服务器拉取
// CACHED 模式：使用缓存（最多 1 小时过期）

// 企业代理：
// 大型企业不允许直连外部服务
// 支持配置内部代理转发 GrowthBook 请求
```

## remoteManagedSettings：远程托管设置

### 架构

```typescript
// 组织管理员可以远程控制 Claude Code 的某些设置
// 用于企业部署的统一配置管理

// 轮询间隔：5 分钟
// 缓存：ETag 条件请求（节省带宽）
// 覆盖优先级：remoteManagedSettings > userSettings > defaults
```

### 危险设置安全对话框

```typescript
// 某些设置被标记为"危险"（如自动接受所有权限）
// 管理员通过远程托管设置推送危险设置时，
// 用户会看到安全警告对话框：
// "您的组织管理员已启用 [设置名]，这会 [危险描述]"
// 用户可以拒绝并将问题上报给管理员

const DANGEROUS_SETTINGS = [
  'bypassPermissions',    // YOLO 模式
  'skipPermissionChecks', // 跳过权限检查
] as const
```

### ETag 条件请求

```typescript
// 设置轮询使用 HTTP ETag 缓存：
// 1. 首次请求：服务器返回设置 + ETag
// 2. 后续请求：带 If-None-Match: {etag} 请求头
// 3. 未变更：服务器返回 304 Not Modified（无 body）
// 4. 已变更：服务器返回新设置 + 新 ETag
// 节省带宽：大多数轮询返回 304
```

## grove：服务条款与隐私同意

```typescript
// Grove = Anthropic 条款/隐私同意管理系统
// 用户首次使用前必须接受

// 设计特点：
// 1. 24 小时缓存（防止每次启动都查询）
// 2. 非阻塞首次加载：启动时异步查询，不等待
// 3. 如果查询失败，使用缓存中的上次状态
// 4. 如果没有缓存，允许通过（fail-open）

// 原因：条款查询失败不应阻塞用户工作
// 条款状态变更（如用户撤销同意）通过下次成功查询生效
```

## 反常规设计

| 模式 | 原因 |
|------|------|
| Datadog Token 硬编码 | 写入 Token 设计为公开；读取 Token 受保护 |
| GrowthBook 缓存导致 worktree bug | CACHED_MAY_BE_STALE 首次为空，已迁移到始终启用 |
| Kill-switch 失败开放 | 分析服务中断不应停止 kill-switch 功能 |
| ETag 条件请求 | 设置很少变更；304 响应节省大量带宽 |
| Grove 非阻塞首次加载 | 条款查询延迟不应让用户等待 |
| 危险设置需要用户确认 | 组织管理员不应能静默启用破坏性设置 |
| 品牌类型作为文档 | `I_VERIFIED_THIS_IS_NOT_CODE_OR_FILEPATHS` 强制开发者举证 |
