## 语音 STT 服务（voice.ts）

### 平台检测与 WSL2 支持

```typescript
// 语音功能需要麦克风访问权限
// 不同平台的实现差异：

// macOS：原生 WebKit/WebView 麦克风 API
// Linux（直接）：ALSA 或 PulseAudio
// WSL2 特殊处理：
//   WSL2 无法直接访问 Windows 麦克风
//   需要通过 PulseAudio over TCP 桥接：
//   Windows 端：运行 PulseAudio 服务器
//   WSL2 端：设置 PULSE_SERVER=tcp:localhost 环境变量
//   Claude Code 检测到 WSL2 时自动配置

function isWSL2(): boolean {
  return process.env.WSL_DISTRO_NAME !== undefined
      || fs.existsSync('/proc/sys/fs/binfmt_misc/WSLInterop')
}
```

### voiceStreamSTT WebSocket 协议

```typescript
// 语音流式传输协议：
// 1. 客户端建立 WebSocket 连接
// 2. 客户端持续发送音频帧（PCM 16bit, 16kHz）
// 3. 服务端（Deepgram Nova 3）实时返回中间识别结果
// 4. 检测到静音（300ms 无声音）时发送最终结果
// 5. 最终结果触发文本输入提交

// voiceKeyterms：用户自定义术语表
// 解决语音识别对专业术语的识别问题
// 例如："kubectl" 可能被识别为 "cute cuddle"
// 用户可以在设置中添加自定义词汇表提高准确率
```

### 确定性 UUID 脱水/水合

```typescript
// 问题：fixture 中的 UUID 是随机的（不可重现）
// 解决：脱水（dehydrate）时将 UUID 替换为确定性 ID

// 脱水算法：
const uuidMap = new Map<string, string>()
let counter = 0

function dehydrateUuid(uuid: string): string {
  if (!uuidMap.has(uuid)) {
    uuidMap.set(uuid, `test-uuid-${counter++}`)
  }
  return uuidMap.get(uuid)!
}
// 相同 UUID 总是映射到相同的 test-uuid-N
// 不同 UUID 映射到不同的 test-uuid-N
// 顺序由首次出现的顺序决定（确定性）

// 水合（hydrate）：将 test-uuid-N 替换回真实 UUID（反向映射）
```

### fixture 脱水字段

```typescript
// 除 UUID 外，以下字段也会被脱水：
const DEHYDRATED_FIELDS = [
  'file_path',      // 替换为相对路径（避免绝对路径差异）
  'cost_usd',       // 替换为固定值（成本随时间变化）
  'duration_ms',    // 替换为固定值（执行时间不确定）
  'created_at',     // 替换为固定时间戳
  'session_id',     // 替换为确定性 ID
]
```

## AgentSummary 服务

### 共享缓存架构

```typescript
// AgentSummary 每 30 秒更新一次任务状态摘要
// 问题：如果多个 Agent 同时请求摘要，会发送重复请求

// 解决：通过 cacheSafeParams 共享缓存
// cacheSafeParams = 确定性可哈希的参数集
// 相同参数的请求共享同一个缓存条目
// 即使来自不同 Agent 实例

type CacheSafeParams = {
  agentId: string
  lastMessageId: string
  // 不包含时间戳！否则缓存无效
}
```

### 🔥 canUseTool 拒绝所有工具的悖论

```typescript
// AgentSummary 生成时包含完整的工具 schema
// 但 canUseTool() 检查会拒绝所有工具调用
// 结果：模型能看到工具定义，但永远不能调用它们

// 为什么这样设计？
// 包含工具 schema：让模型理解 Agent 的能力范围
// 拒绝工具调用：摘要应该只是"总结"，不应该执行操作

// 这是有意的约束：知道工具存在 ≠ 可以使用工具
```

## preventSleep（防休眠服务）

### macOS caffeinate 自愈机制

```typescript
// 长时间运行的任务（如 Agent 工作）需要防止 macOS 休眠
// 使用 macOS 的 caffeinate 命令

// 关键参数：caffeinate -i -t 300
// -i：防止系统空闲休眠
// -t 300：最多 5 分钟后自动失效（安全超时）

// 自愈机制：
// caffeinate 每 5 分钟自动过期
// 但在过期前 1 分钟（4 分钟时），重新启动新的 caffeinate
// 确保连续防休眠，同时避免永久锁定

const CAFFEINATE_TIMEOUT = 300      // 5 分钟
const RESTART_BEFORE_EXPIRY = 60    // 过期前 1 分钟重启

// 引用计数：
// 多个工具/任务可能同时需要防休眠
// 使用引用计数管理：
// refCount > 0：维持 caffeinate
// refCount = 0：停止 caffeinate
```

## 反常规设计

| 模式 | 原因 |
|------|------|
| Tips 通过 GrowthBook A/B 测试 | 不同描述对用户行为影响显著；A/B 测试找到最有效表达 |
| WSL2 PulseAudio 桥接 | WSL2 无法直接访问麦克风；TCP 桥接是唯一方案 |
| VCR UUID 确定性脱水 | 随机 UUID 导致 fixture 每次录制都不同，无法版本控制 |
| VCR 成本/时间字段脱水 | 成本和执行时间随时间变化；测试需要稳定的值 |
| AgentSummary 含工具 schema 但拒绝调用 | 知道工具存在（理解能力范围）≠ 被允许调用工具 |
| caffeinate 5 分钟超时 + 4 分钟重启 | 避免崩溃时永久阻止 Mac 休眠；自愈保证连续性 |
| TODO(human) 占位符 | 标记"需要人类判断"的实现决策，不应由 AI 随意填充 |
