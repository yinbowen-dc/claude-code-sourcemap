### 迁移设计原则

```typescript
// 迁移使用幂等状态检查设计
// 每次迁移先检查是否已完成，再执行

async function migrateModelName(config: Config): Promise<void> {
  // 1. 读取当前配置中的模型名
  const current = config.model

  // 2. 检查是否是旧模型名（幂等检查）
  if (!isOldModelName(current)) return  // 已经迁移过，跳过

  // 3. 映射到新模型名
  const newModel = MODEL_NAME_MAP[current]

  // 4. 写回配置
  await config.set({ model: newModel })
}
```

## bootstrap/state.ts：启动状态快照

### 规模

```
257 个状态字段
涵盖：用户配置、会话状态、UI 状态、权限状态、缓存状态、
     遥测状态、工具状态、模型配置、API 配置...
```

### 关键设计：Prompt 缓存稳定性字段

```typescript
// 两个特殊的"锁存"字段，保持提示缓存稳定：

afkModeHeaderLatched: boolean
// AFK 模式下，系统提示头部被锁定（不变）
// 即使用户返回，也不立即改变提示结构
// 原因：改变提示结构会破坏 Anthropic 的提示缓存
// 等到下一个自然边界才解锁

fastModeHeaderLatched: boolean
// 快速模式的系统提示头部锁定
// 同理：防止模式切换期间的提示缓存破坏
// 提示缓存命中 = 节省 token 费用 + 降低延迟
```

### 状态快照机制

```typescript
// 所有 257 个字段都可以序列化
// 快照保存到：~/.claude/projects/{hash}/state.json
// 会话恢复时加载快照，实现无缝续接

// 部分字段不持久化（运行时计算）：
// - 当前活跃的 WebSocket 连接
// - 内存中的工具实例
// - 临时缓存
```

## native-ts：原生 TypeScript 移植模块

三个精心移植的 JavaScript 原生库，完全用 TypeScript 重写，
避免对外部 npm 包的依赖，允许在受限环境（如 Bun）中运行。

### color-diff（来自 highlight.js）

```typescript
// 原始库：highlight.js 的颜色差异计算
// 用途：在终端中高亮显示代码差异

// 关键特性：懒加载语法高亮
// 不在启动时加载所有语言语法，只在需要时加载
// 大幅减少启动时间

// ANSI 256 色支持：
// 将 CSS 颜色（#RRGGBB）映射到最接近的 ANSI 256 色码
// 使用感知色彩空间（Lab 颜色空间）计算"最近色"
// 而不是简单的 RGB 欧氏距离（视觉更准确）
```

### file-index（来自 nucleo 模糊搜索）

```typescript
// 原始库：nucleo（Rust 编写的模糊搜索引擎）
// 用途：文件名模糊搜索（如 Ctrl+P）

// 单次扫描 Top-K 堆算法：
// 不先收集所有结果再排序（O(n log n)）
// 而是维护一个大小为 K 的最大堆
// 扫描时实时淘汰低分结果（O(n log k)）
// 对于 K 很小时（通常 K=20）大幅提速

// 评分函数考虑：
// - 首字母匹配（大幅加分）
// - 连续匹配（加分）
// - 驼峰边界匹配
// - 路径分隔符后的匹配
```

### yoga-layout（来自 Meta 的 Flexbox 引擎）

```typescript
// 原始库：facebook/yoga（C++ Flexbox 布局引擎）
// 用途：Ink（React 终端框架）的布局计算

// 规模：2578 行 TypeScript
// 完整移植 Flexbox 算法

// 🔥 NaN 安全相等比较：
// Yoga 中浮点数可能是 NaN（未定义尺寸）
// JavaScript 中 NaN !== NaN（IEEE 754 规范）
// 因此不能用 === 比较节点状态变化
// 使用自定义 isEqual(a, b)：
function isEqual(a: number, b: number): boolean {
  if (isNaN(a) && isNaN(b)) return true  // 两个 NaN 视为相等
  return a === b
}
// 没有这个，布局引擎会把"两个未定义尺寸"视为不同，
// 触发无限重新布局循环
```

## 反常规设计

| 模式 | 原因 |
|------|------|
| 迁移幂等状态检查 | 迁移可能被意外运行多次；检查防止重复迁移 |
| afkModeHeaderLatched 字段 | 模式切换不立即改变提示结构，保护提示缓存 |
| 257 字段全序列化快照 | 确保会话可以在任意点中断并完整恢复 |
| 移植 yoga-layout 而非用 wasm | Bun 环境加载 wasm 比 TypeScript 慢；纯 TS 可被编译器优化 |
| NaN 安全相等（yoga-layout） | JavaScript NaN !== NaN 会导致无限重布局 |
| file-index Top-K 堆 | 用户通常只需要前 20 个结果，Top-K 比全排序快得多 |
| moreright 存根模式 | 将公开代码库和内部能力解耦，同一代码库两种编译产物 |
