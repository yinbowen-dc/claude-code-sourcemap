# AI 自动化后台服务

## 概述

Claude Code 在后台运行多个 AI 驱动的自动化服务：记忆整合（autoDream）、
文档自动维护（MagicDocs）、补全预测（PromptSuggestion/Speculation）、以及离开摘要（awaySummary）。

**相关文件：** `services/autoDream/`、`services/MagicDocs/`、`services/PromptSuggestion/`、
`services/awaySummary/`

---

## autoDream：背景记忆整合

### 概述

autoDream 在会话空闲时自动将对话内容整合进长期记忆，无需用户手动触发。

### 三重门控系统

```typescript
// 只有三个条件同时满足，才触发整合：

// 1. 时间门控：距上次整合超过 24 小时
const TIME_GATE = 24 * 60 * 60 * 1000  // 24h

// 2. 会话门控：至少积累了 5 个新会话
const SESSION_GATE = 5

// 3. 锁文件门控：无其他进程正在运行整合
// 用 lockfile 包实现互斥（同项目只运行一个实例）
```

### 🔥 mtime 即 lastConsolidatedAt 的锁技巧

```typescript
// 不使用单独的时间戳文件来记录"上次整合时间"
// 而是直接读取锁文件的 mtime（文件修改时间）
// 锁文件创建时间 = 整合开始时间

// 为什么巧妙：
// 1. 锁文件必须存在才能运行（互斥）
// 2. 锁文件的 mtime 天然记录了上次运行时间
// 3. 一个文件同时承担两个职责
// 如果整合崩溃，锁文件仍在，防止立即重试
```

### 整合流程

```typescript
// 1. 扫描所有项目记忆文件
// 2. 扫描最近 N 个会话记录
// 3. 使用 claude-sonnet 生成整合摘要
// 4. 更新记忆文件，合并重复项
// 5. 释放锁（更新 mtime）
```

## MagicDocs：自动文档维护

### 概述

MagicDocs 自动维护带有 `# MAGIC DOC:` 头部的特殊文档文件。
当 Claude 完成一轮操作并进入空闲时，自动更新这些文档。

### 触发机制

```typescript
// 触发条件：Claude 空闲（用户轮次结束后）
// 检测方式：idleTurn 事件
// 不在每次工具调用后触发——只在完整轮次结束后

// 空闲检测：
onIdleTurn(() => {
  if (hasMagicDocs()) {
    scheduleMagicDocsUpdate()
  }
})
```

### MAGIC DOC 文件格式

```markdown
# MAGIC DOC: 项目架构图

<!-- 以下内容由 Claude 自动维护 -->
<!-- 上次更新：2026-03-28T10:30:00Z -->

[自动生成的内容...]
```

检测逻辑：读取文件第一行，检查是否以 `# MAGIC DOC:` 开头。

### 批量更新策略

```typescript
// 项目中可能有多个 MAGIC DOC 文件
// 并发更新（不串行）——但限制并发数防止 Token 爆炸
// 更新失败时静默忽略（不影响主对话流）
```

## PromptSuggestion / Speculation（推测性预填充）

### 概述

当用户输入时，在后台推测性地预运行可能的补全，以减少最终触发时的延迟。

### 写时复制沙盒（Copy-on-Write Overlay）

```typescript
// 推测执行必须不影响真实状态
// 使用写时复制（CoW）覆盖层隔离推测状态：
const SPECULATION_BASE = '~/.claude/temp/speculation/{PID}/{id}'

// 推测沙盒：
// 1. 读操作：先查沙盒，找不到则读真实文件系统
// 2. 写操作：写到沙盒，不修改真实状态
// 3. 丢弃：用户选择其他路径时，直接删除沙盒目录
// 4. 提交：用户确认时，将沙盒变更合并到真实状态
```

### 缓存键隔离（PR #18143 事后教训）

```typescript
// 历史 Bug（PR #18143）：
// effort:'low' 推测共享了同一个缓存键
// 结果：多个推测竞争同一缓存条目
// 现象：缓存写入量激增 45 倍

// 修复：每个推测实例有唯一缓存键
// 缓存键 = 基础参数 + speculationId（UUID）
// 推测完成后，缓存条目随沙盒一起清理
```

### 管道化补全

```typescript
// 推测执行流水线：
// 1. 检测用户输入变化（防抖 ~200ms）
// 2. 生成候选补全参数
// 3. 在 CoW 沙盒中预执行
// 4. 缓存结果，等待用户确认
// 5. 确认：直接返回缓存结果（0 延迟）
// 6. 拒绝：丢弃沙盒，重新执行
```

## awaySummary：离开摘要服务

### 用途

当用户长时间离开（AFK）时，自动生成"你不在时发生了什么"的摘要。

```typescript
// 触发：检测到用户长时间无输入
// AFK 阈值：从 state.afkModeHeaderLatched 标志检测

// 摘要内容：
// - 完成的工具调用
// - 产生的文件变更
// - 遇到的错误
// - 需要用户决策的事项

// 展示：在用户返回时显示在顶部
```

### 与 AgentSummary 的区别

| 服务 | 触发时机 | 目标受众 |
|------|---------|---------|
| awaySummary | 用户 AFK 后返回 | 人类用户快速了解进展 |
| AgentSummary | 子 Agent 完成任务 | 协调器 Agent 了解子 Agent 结果 |

## 反常规设计

| 模式 | 原因 |
|------|------|
| 三重门控（时间+会话+锁） | 防止过度触发；任何单一条件都不足以触发整合 |
| mtime 作为时间戳 | 锁文件天然记录最后运行时间，无需额外状态文件 |
| MagicDocs 仅空闲时触发 | 避免工具调用期间的文档更新干扰主流程 |
| CoW 沙盒路径含 PID | 进程崩溃时，操作系统重用 PID 不会与旧沙盒冲突 |
| 推测缓存键含 speculationId | PR #18143 教训：共享缓存键导致 45x 缓存写入激增 |
| 推测失败静默丢弃 | 推测只是优化，失败不应影响主交互质量 |
