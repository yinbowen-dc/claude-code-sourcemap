# 查询引擎与上下文管道

## 概述

两层 AI 编排架构：`QueryEngine`（会话生命周期）包裹 `query()`（单次请求循环）。

**相关文件：** `QueryEngine.ts`、`query.ts`、`query/`、`context.ts`、`context/`、
`history.ts`、`utils/systemPrompt.ts`、`utils/truncate.ts`、`utils/transcriptSearch.ts`

---

## QueryEngine（QueryEngine.ts）

每个对话一个实例。管理跨轮次的可变状态。

```
submitMessage(用户输入):
  1. 初始化轮次作用域状态
  2. 获取系统提示各部分（默认 + 用户上下文 + 系统上下文）
  3. 处理用户输入（斜杠命令、附件、模型覆盖）
  4. 立即持久化消息 → 崩溃时也能 --resume
  5. 构建完整系统提示
  6. 发送系统初始化消息
  7. 如果只有本地命令 → 提前返回（不调用 API）
  8. 委托给 query() 循环
  9. 收尾：累计用量、追踪费用
```

### 消息持久化时机

```typescript
// 用户消息：API 调用前同步写入
await recordTranscript(messages)

// 助手消息：即发即忘
void recordTranscript(messages)  // 异步：message_delta 可能修改最后一块内容
```

原因？100ms 的 `jsonStringify` 窗口可以吸收流式传输块带来的修改。

### 跨事件的用量追踪

```
message_start  → 初始 input_tokens
message_delta  → 每块更新（stop_reason 在这里到达）
message_stop   → 最终化，加入会话总计
```

`stop_reason` 只在 `message_delta` 中到达，不在 `content_block_stop` 中。

## query() 函数（query.ts）

纯异步生成器：每次 API 请求循环一个。向 QueryEngine 返回事件流。

### 恢复流程

查询循环通过状态机实现 **7+ 种恢复流程**：

| 触发条件 | 恢复动作 |
|---------|---------|
| 提示过长 | 1. 上下文折叠排空 → 2. 响应式压缩 → 3. 报告错误 |
| 最大输出 Token 超限 | 1. 升级模型 → 2. 重试 → 3. 报告错误 |
| 停止钩子阻塞 | 注入钩子反馈后重试 |
| Token 预算 | 插入继续消息，恢复执行 |
| 工具执行错误 | 将错误注入为工具结果，继续 |
| 媒体不可剥离 | 报告错误 |
| 阻塞限制 | 合成 'invalid_request'，退出 |

**错误扣留：** 循环扣留错误，直到所有恢复尝试都用尽才报告给用户，
防止可自动恢复的瞬态错误被误报。

### 并发工具执行

```typescript
// 使用 StreamingToolExecutor：
// 模型流式传输期间并行获取工具结果（5-30 秒重叠窗口）
// 不使用时：流式传输结束后批量处理（更慢，阻塞下一轮）
```

### 延迟工具摘要生成

```typescript
// 在下一次模型流式传输期间即发即忘地触发 haiku 摘要调用
// 重叠：haiku（约 1s）+ 模型流（约 5-30s）= 零关键路径影响
// 模型完成时在待处理缓冲区中等待
```

## 上下文组装

三个独立的、记忆化的上下文来源：

### 1. 用户上下文（context.ts）

每个对话记忆化一次，前置到 `messages[0]`。

```typescript
{
  claudeMd: string,    // 从工作目录遍历发现的 CLAUDE.md 文件
  currentDate: string  // ISO 格式日期
}
```

以下情况禁用 CLAUDE.md 发现：
- `CLAUDE_CODE_DISABLE_CLAUDE_MDS=1`
- `--bare` 模式且未使用 `--add-dir`

通过 `setCachedClaudeMdContent()` 缓存，供 yoloClassifier 访问。

### 2. 系统上下文（context.ts）

每个对话记忆化一次，追加到系统提示。

```typescript
{
  gitStatus: string,    // 分支、差异（限 2000 字符）、最近提交
  cacheBreaker: string  // 临时调试注入（仅 ant，特性开关控制）
}
```

以下情况跳过 git 信息：`CLAUDE_CODE_REMOTE=1` 或 git 说明被禁用。

### 3. 工具使用上下文

每个 QueryEngine 在内存中维护，流过 query 循环的 `ToolUseContext`：
- `tools`、`mainLoopModel`、`abortController`、`readFileState`、`getAppState()`

## 系统提示层级（systemPrompt.ts）

优先级（首个匹配生效）：

```
1. overrideSystemPrompt   → 替换所有内容（循环/非交互模式）
2. coordinator 提示       → 如果启用了 COORDINATOR_MODE
3. agent 提示             → 在主动模式下：追加到默认提示
4. agent/custom/default   → 非主动模式：agent 替换默认提示
5. appendSystemPrompt     → 始终追加到末尾
```

`SystemPrompt` 是品牌类型（`readonly string[] & { __brand: 'SystemPrompt' }`），
在编译时防止误传原始字符串。

**模型实际接收的完整顺序：**
1. 系统提示（来自上述层级）
2. + 系统上下文（git 状态）
3. + messages[0]（当前日期 + 记忆文件）

## 对话历史

### 会话记录（Transcript）

每个项目一个只追加 JSONL 文件：`~/.claude/projects/{hash}/transcript.jsonl`

条目类型：`TranscriptMessage`、`ContentReplacementEntry`、`CompactEntry`、`SnipEntry`

**内容替换追踪：** 当大型工具结果被替换为预览时，
`ContentReplacementEntry` 记录映射关系。**在分叉/创建分支时保留**，
防止分叉会话中的提示缓存预算不匹配。

### 输入历史（history.ts）

全局文件：`~/.claude/history.jsonl`

```typescript
type LogEntry = {
  display: string     // 选择器的单行显示文本
  pastedContents: Record<number, StoredPastedContent>
  timestamp: number
  project: string     // 提交时的工作目录
  sessionId?: string
}
```

检索顺序：
1. 当前会话条目优先
2. 其他会话，按时间倒序
3. 最多 100 条

粘贴内容处理：
- ≤1024 字符：内联存储
- >1024 字符：哈希引用，单独存储（即发即忘异步写入）

## 截断（truncate.ts）

所有函数都是**终端宽度感知**且**CJK/emoji 安全**的（通过 `stringWidth()`）：

| 函数 | 行为 |
|------|------|
| `truncatePathMiddle(path, width)` | 保留目录上下文 + 文件名：`src/…/MyComponent.tsx` |
| `truncateToWidth(str, width)` | 从末尾截断，追加 `…` |
| `truncateStartToWidth(str, width)` | 从开头截断，前置 `…` |
| `truncateToWidthNoEllipsis(str, width)` | 截断但不加省略号 |
| `wrapText(str, width)` | 按宽度分割为多行 |

git 状态上限 2000 字符：
```typescript
status.substring(0, 2000) + '\n... (truncated. Run "git status" using BashTool)'
```

## 反常规设计

| 模式 | 原因 |
|------|------|
| 错误扣留直到恢复耗尽 | 防止将可自动恢复的瞬态错误提前暴露给用户 |
| 每条消息的工具结果预算在 microcompact 之前执行 | 两个系统干净组合；microcompact 永远不检查内容 |
| Snip 边界在 mutableMessages 上重放 | 不删除的话，标记每次轮次都会重新触发（内存泄漏） |
| 助手消息用 `void recordTranscript()` | 100ms jsonStringify 窗口吸收流式传输的修改 |
| 跨多个事件累计用量 | `stop_reason` 只在 `message_delta` 中到达 |
| SystemPrompt 品牌类型 | 编译时防止误传原始字符串 |
| 分叉时保留内容替换数据 | 防止分叉会话中提示缓存预算不匹配 |
| 即发即忘的 haiku 摘要 | 与模型流重叠 = 零关键路径影响 |
